# amp-app-banner 実装メモ

*date:2018-12-10*

*desc> AMPシリーズ*

## はじめに
amp-app-banner という、AMP サイトでアプリのダウンロードを促すためのバナーのコンポーネントを実装しました。  
自分の活動記録を兼ねて、現状の挙動についてのメモを残しておこうと思います。

## できるもの
https://ampbyexample.com/components/amp-app-banner/ このページそのものがデモになっています。スマートフォンでこのページにアクセスすると挙動が確認できます。各ブラウザのバージョンは最新前提です。執筆段階で iOS - Safari は 12、Chrome は 71.xxxx。

## 挙動について
現状の OS / ブラウザでは先に挙げた[このページ](https://ampbyexample.com/components/amp-app-banner/)下部の動画の通りにはいかないようです。動画の下の Testing という項目でブラウザの開発者パネルでのエミュレータで設定できるカスタム・ユーザー・エージェントに `iPhone; CPU iPhone OS 9_1` とか書いてあるし、その頃の実装なのでしょうか（実際その設定をするとたしかに動画同様のコンポーネント表示が確認できる）。

現状の最新 OS / 最新ブラウザに関しては、iOS - Safari / Android - Chrome だとブラウザ標準みたいなバナーが、Safari だと上部、Android - Chrome だと下部に表示されますね。  
逆に、AMP コンポーネントデザインが表示されるパターンが iOS - Chrome / Android - Firefox でした。

| - | iOS - Safari | iOS - Chrome | Android - Chrome | Android - Firefox | Android - 標準ブラウザ |
|:-----------|:-----------|:------------|:------------|:------------|:------------|
| バナーデザイン | ブラウザ標準 | AMP デザイン | ブラウザ標準 | AMP デザイン | AMP デザイン |
| 表示位置 | 上部 | 下部 | 下部 | 下部 | 下部 |
| APP インストール時 | インストール時表示 | インストール時表示 | インストール時非表示 | インストール時表示 | インストール時非表示 |

何気に「Android - 標準ブラウザ」という項目もありますが、リンクタップでの遷移先で「対応してません」という表示が出ますね。
